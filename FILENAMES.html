🌸 Bloom Builder — Image Filename Reference

Keep this page open while you upload images to GitHub. Every image file must be named exactly as shown below.

Steps to replace any flower or ribbon image:
  1. Prepare your image. PNG or JPG, ideally square, under 500 kb. Free resizer: squoosh.app
  2. Rename the file to exactly match the Filename column below (including the .png or .jpg extension).
  3. Go to your GitHub repository → open (or create) the images folder → click Add file → Upload files → drag your renamed file in → click Commit changes.
  4. Open data.js in GitHub → click the pencil (edit) icon → find the flower by name → add this line right after its "origin" line:
        imgUrl: "images/rose-red.png", (replace rose-red.png with your actual filename) → click Commit changes.
  5. Wait about 60 seconds → refresh the website. Your image is live for everyone.

To go back to the pencil drawing at any time: delete the imgUrl line in data.js and commit.

📁 What your GitHub repository should look like

your-repo/
├── index.html
├── theme.css
├── data.js
├── FILENAMES.html
└── images/
    ├── logo.png          ← your logo (shows in top-left header)
    ├── background.png    ← canvas paper texture (optional)
    ├── snigdha.png       ← the secret flower photo
    ├── rose-red.png
    └── ... (only upload the ones you want to replace)

Logo: name it exactly logo.png and upload to the images/ folder. It appears in the top-left corner of the header automatically.

Background: you can upload any image from the left panel of the website directly — it applies immediately to the canvas for that session. For a permanent background, name it background.png and upload to images/, then add one line to index.html's canvas style. (Or just use the in-app upload each time you open it.)

You don't need to upload all flower images — only upload the ones you want to replace. Everything else shows its pencil drawing.
